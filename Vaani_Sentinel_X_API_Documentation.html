5. ‚ùå Error Handling

HTTP Status Codes

Status Code Meaning When It Occurs
200 OK Successful request
400 Bad Request Missing required fields, invalid JSON
401 Unauthorized Missing or invalid authentication
404 Not Found Resource not found, invalid endpoint
500 Internal Server Error Server-side error

Error Response Format

{ "error": "Error message", "message": "Detailed description", "timestamp": "2025-07-31T09:28:01.007Z" }

Common Error Examples

// Missing token { "error": "Access token required" } // Invalid credentials { "error": "Invalid credentials" } // Resource not found { "error": "Translated content not found" } // Rate limit exceeded { "error": "Too many requests from this IP, please try again later." }

6. üîí Security & Rate Limiting

Rate Limiting

  • Scope: All /api/* endpoints
  • Limit: 100 requests per 15 minutes per IP
  • Window: Rolling 15-minute window
  • Response: 429 status code when exceeded

Security Features

  • JWT-based authentication with 24-hour expiry
  • Security headers (Helmet.js)
  • CORS protection
  • Request size limits (10MB)
  • Input validation and sanitization

CORS Configuration

{ "origin": ["http://localhost:3000", "http://localhost:3001"], "credentials": true, "methods": ["GET", "POST", "PUT", "DELETE", "OPTIONS"], "allowedHeaders": ["Content-Type", "Authorization"] }
